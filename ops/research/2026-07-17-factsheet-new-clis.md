# OpenCode / Kimi Code / Cursor CLI fact sheet — verified July 17, 2026

Researched by newcli-researcher from official docs. VERIFIED/UNVERIFIED tags per claim. Upfront flags: open-code.ai (hyphenated) is a DECOY domain — official is opencode.ai. Kimi has two products: legacy "Kimi CLI" (PyPI, winding down) and current **Kimi Code CLI** (github.com/MoonshotAI/kimi-code).

## 1. OpenCode
- (a) Org **anomalyco** (renamed from sst/opencode). Docs: opencode.ai/docs. Install: curl opencode.ai/install | bash, npm i -g opencode-ai, brew install anomalyco/tap/opencode. Latest **v1.18.3** (July 16, 2026). VERIFIED.
- (b) Headless: `opencode run "prompt"`, `--model provider/model`, `--format json` (raw JSON events). `--attach http://localhost:4096` to a running `opencode serve` (skips cold-boot). Also HTTP API server + ACP server. Exit codes NOT documented (gap). VERIFIED (opencode.ai/docs/cli).
- (c) Agents: global ~/.config/opencode/agents/, project .opencode/agents/; filename = id. Markdown+YAML: description (req), mode (primary/subagent/all), model, prompt, temperature, top_p, steps, disable, hidden, color, **permission** (per-tool allow/ask/deny map). Built-ins: build (full), plan (read-only), general/explore/scout subagents. VERIFIED (docs/agents).
- (d) Permissions: `--auto` auto-approves anything not explicitly denied; per-tool per-pattern rules in opencode.json (tools: read, edit, bash, external_directory, glob, grep, task, skill, lsp, question, webfetch, websearch). VERIFIED (docs/permissions).
- (e) Models: BYOK via /connect, 75+ providers (AI SDK + Models.dev) + local. No hardcoded default; docs recommend GPT 5.2, Claude Opus 4.5/Sonnet 4.5, Gemini 3 Pro. Reasoning effort varies by provider: OpenAI none→xhigh, Anthropic high(default)/max, Google low/high. VERIFIED (docs/models).
- (f) AGENTS.md supported. Skills: SKILL.md/agentskills.io standard from .opencode/skills, .claude/skills, **.agents/skills** + global equivalents. VERIFIED (docs/skills). → Triforge's existing .agents/skills interop (ADR D-003) pays off directly.
- (g) Hooks: rich plugin-event system (tool.execute.before/after, permission.asked/replied, session.*, file.edited, command.executed, message.*, tui.*) via opencode.json "plugin" or .opencode/plugins/. VERIFIED (docs/plugins). Headless firing of non-TUI events UNVERIFIED (likely yes; confirm).
- (h) Fragility: headless device-code auth for SSH/Docker/CI exists. No verified rate-limit/telemetry issues in core.

## 2. Kimi Code CLI (current; supersedes legacy kimi-cli)
- (a) Moonshot AI, github.com/MoonshotAI/kimi-code. Install: curl code.kimi.com/kimi-code/install.sh | bash, brew install kimi-code, npm. Latest **@moonshot-ai/kimi-code@0.27.0** (July 17, 2026). **Near-daily release cadence** — pinned versions short-lived. VERIFIED.
- (b) Headless: `kimi -p "<prompt>"` single turn; **stdout = assistant text only, stderr = thinking/tool progress** (clean separation). `--output-format text|stream-json` (only with -p). Exit codes 0/1 confirmed for login only, convention implied elsewhere. VERIFIED (kimi.com/code/docs kimi-command).
- (c) Built-in subagents: coder (v0.27: background tasks, todo lists, plan mode, skill invocation, nested agents), explore, plan — isolated contexts. AGENTS.md: $KIMI_CODE_HOME/AGENTS.md (~/.kimi-code/), cross-tool ~/.agents/AGENTS.md, project .kimi-code/AGENTS.md or AGENTS.md. VERIFIED. **Custom agent definitions (--agent-file YAML/markdown) documented only for legacy kimi-cli — UNVERIFIED for kimi-code.**
- (d) `-p` mode auto-uses **auto** permission policy (no prompts); static config deny rules still enforced. No granular per-tool allowlist flag found; no --sandbox equivalent (gap). VERIFIED.
- (e) Models: **Kimi K2.7 Code** (coding-focused, ~30% fewer thinking tokens than K2.6, requires Thinking On — kimi.com/resources/kimi-k2-7-code) and **Kimi K3** (flagship, 2.8T params, KDA hybrid attention, native vision, 1M context; reasoning = "max effort" only, finer levels planned — kimi.com/blog/kimi-k3). Flag: `-m`, e.g. `kimi -m kimi-code/kimi-for-coding -p`. Auth API-key only (sk-…); subscription auth NOT available (GH kimi-cli#757). VERIFIED.
- (f) AGENTS.md confirmed; SKILL.md/agentskills.io interop UNVERIFIED (skill invocation exists per changelog, path/standard unconfirmed).
- (g) Hooks: repo mentions "lifecycle hooks" to gate risky tool calls / audit — no event-list docs reached. UNVERIFIED/thin.
- (h) Fragility: telemetry ON by default (opt out KIMI_DISABLE_TELEMETRY). Rate limits tiered by top-up (429; ~1000 req/min example tier — platform.kimi.ai/docs/pricing/limits). Near-daily releases = main stability risk.

## 3. Cursor CLI
- (a) Binary `cursor-agent` (docs shorthand: agent), Anysphere, closed-source. Docs cursor.com/docs/cli. Install: curl cursor.com/install -fsS | bash. **No published semver** — changelog is date-based (latest "July 6, 2026"); pin via `agent --version` post-install. Auto-updates by default (drift risk). VERIFIED.
- (b) Headless: `agent -p/--print`, `--output-format text|json|stream-json`, `--stream-partial-output`. `-f/--force` (alias --yolo) applies edits without confirmation. Exit codes via $? examples, not enumerated. VERIFIED (docs/cli/headless).
- (c) Agents: project .cursor/agents/, global ~/.cursor/agents/. Markdown+YAML: name (filename), description (delegation trigger), model (inherit|fast|explicit id), readonly (bool). Background subagents write progress to ~/.cursor/subagents/. Per-agent tools allowlist beyond readonly UNVERIFIED. VERIFIED (docs/subagents).
- (d) Permissions: `-f/--force`, `--sandbox <enabled|disabled>`, `--approve-mcps`, `--trust` (bypass workspace-trust prompt — needed for clean headless). VERIFIED (docs/cli/reference/parameters).
- (e) Models: `--model`, `--list-models`; auth --api-key / CURSOR_API_KEY / agent login. Default ambiguous: docs foreground **Auto** router; third-party blogs claim Composer 2.5 default (unconfirmed — live conflict). Catalog (cursor.com/docs/models): Claude Opus 4–4.7, Sonnet 4–4.6, Haiku 4.5, **Fable 5, Sonnet 5**; GPT-5 series to 5.6; Gemini 2.5/3/3.1/3.5; Grok 4.5; GLM 5.2; Kimi K2.7 Code; Composer 1/2.5. **No reasoning-effort flag found** (gap). VERIFIED.
- (f) Rules: .cursor/rules/ + AGENTS.md + **CLAUDE.md** at root. Skills: SKILL.md, explicitly Claude-Code-compatible, path .cursor/skills/ (project-scoped only, no global dir). VERIFIED (docs/context/skills).
- (g) Hooks: hooks.json (version + event→command). Events: beforeSubmitPrompt, beforeShellExecution, afterFileEdit, stop + July additions (afterAgentResponse, afterAgentThought, subagentStart). VERIFIED (docs/hooks, changelog). **Headless fragility (community-reported, UNVERIFIED): only beforeShellExecution/afterShellExecution reliably fire under cursor-agent; other events + nested matcher shape don't** (forum.cursor.com/t/…/148316). Same class of problem as Codex exec hooks (ADR D-004).
- (h) Fragility: needs agent login or API key; --trust required headless; no semver pinning; auto-update drift; headless hook gap is the standout embedding risk.

## Net (researcher's synthesis)
All three have real one-shot headless modes with clean output separation (OpenCode run --format json, Kimi -p --output-format stream-json, Cursor -p --output-format stream-json) → mechanically embeddable exactly like Gemini/Codex. OpenCode has the most mature/documented permission + hook system. Cursor headless hooks most fragile, version pinning weakest. Kimi custom-agent + hooks stories are the biggest doc gaps (UNVERIFIED) — direct probe needed if load-bearing.
