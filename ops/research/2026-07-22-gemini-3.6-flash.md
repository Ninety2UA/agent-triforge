# Watch note — Gemini 3.6 Flash release (2026-07-21)

Watch-cycle record (cli-watch methodology: research → gap check → adopt/defer).
Trigger: user report 2026-07-22; primary-source verified same day.

## Release facts (verified)

- **Gemini 3.6 Flash** GA'd 2026-07-21, alongside **Gemini 3.5 Flash-Lite**
  and a limited-access **Gemini 3.5 Flash Cyber** (governments/trusted
  partners only). Claimed vs 3.5 Flash: −17% output tokens, DeepSWE 49% (was
  37%), output price $9.00 → $7.50/M (input steady $1.50/M), knowledge cutoff
  Mar 2026, 1M input context, 64k output cap.
- **No new Pro.** Google explicitly shipped no 3.5/3.6 Pro in this drop
  (teased Gemini 4 instead). Gemini 3.1 Pro remains the latest Pro.
- **Live `agy` catalog check (2026-07-22, agy 1.1.4, `agy models`):**
  `Gemini 3.6 Flash (High|Medium|Low)` now listed; `Gemini 3.5 Flash` variants
  retained; `Gemini 3.1 Pro (High|Low)` still the ONLY Pro entries. Display
  names confirm the `(High)/(Low)` suffix convention carries over to 3.6.

Sources: [TechCrunch](https://techcrunch.com/2026/07/21/google-releases-three-new-gemini-models-but-no-3-5-pro/),

## Decision

**Defer (no adoption into defaults).** The never-Flash-by-default pin (KTD-8,
session-settled, user-directed) and the latest-Pro policy both hold unchanged —
there is no new Pro to move to. Gemini 3.6 Flash is available today as an
explicit per-role opt-in (`model = "Gemini 3.6 Flash (High)"` on an agy-backed
role, or `AGY_MODEL` for a one-off), and the shipped docs now name it as the
concrete opt-in example. Revisit on the next Pro release (watch: Gemini 4 tease)
— that WILL move the pins under the latest-Pro policy.
