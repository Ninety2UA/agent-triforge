# .codex/ — Triforge-shipped Codex project config

`hooks.json` enforces the CHANGELOG audit-trail convention mechanically: on the first `PostToolUse` event of a Codex session it appends one `<timestamp> | codex | tool activity in session` attribution line to `ops/CHANGELOG.md` (a `.claude/codex-changelog.<pid>` marker keeps it to one line per session). Hooks only fire under `codex exec` when `invoke-external.sh` passes `--dangerously-bypass-hook-trust` — Codex does not persist project trust for arbitrary dirs, and Triforge ships and vets these hooks itself (trusted-pipeline posture, same rationale as `approval_policy = "never"`); probe CDX-04 (2026-07-17, codex 0.144.4) verified exec-mode firing, see `ops/decisions/2026-07-18-codex-hooks-under-exec.md`. To disable: delete the project's `.codex/hooks.json` (the helper then omits the bypass flag automatically). `config.toml` disables Codex's auto-memory pipeline (see its inline comments).
